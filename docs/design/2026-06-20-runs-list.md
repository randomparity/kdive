# runs.list — filterable, viewer-gated Run listing

Issue: #623 (`AX_REVIEW.md` A2) · ADR: [0198](../adr/0198-runs-list.md) ·
Depends on: [ADR-0192](../adr/0192-list-pagination-envelope.md) (pagination, merged)

## Problem

`runs.*` has no list tool. A Run is reachable only one-at-a-time by id, so "which Runs
are on System X?" and "what is building across my project?" cannot be answered in one
call.

## Goal

A read-only, viewer-gated `runs.list` filterable by `system_id` / `investigation_id` /
`state`, returning the standard per-item Run envelope plus the ADR-0192 pagination
payload, with the same no-leak project boundary as `systems.list`.

## Non-goals

- No `target_kind` (or any other) filter beyond the three the issue names.
- No per-item enrichment beyond what `envelope_for_run` renders from the Run row alone
  (no `required_cmdline`, failing-job reason, debug sessions, or step progress — those
  stay get-only).
- No schema or migration change.

## Design

### Handler

New module `src/kdive/mcp/tools/lifecycle/runs/list.py`, mirroring
`systems/view.py::list_systems` minus the allocation/resource join:

- `RunsListRequest` (frozen dataclass): `system_id`, `investigation_id`, `state` (all
  `str | None`), `limit: int = DEFAULT_LIST_LIMIT`, `cursor: str | None = None`.
- `list_runs(pool, ctx, request) -> ToolResponse`. **Validation precedes scoping** — a
  malformed filter or cursor is a `configuration_error` regardless of the caller's project
  grants, mirroring `list_systems` (it validates filters at line 250 and decodes the cursor
  at 261 *before* the empty-viewer short-circuit at 267). Returning an empty collection
  first would mask a malformed cursor/filter and make the error path depend on the caller's
  grants:
  1. Compute `viewer_projects = [p for p in ctx.projects if ctx.roles.get(p) is not None]`
     (used in step 4, not yet a return point).
  2. Build filter clauses, starting with `project = ANY(%s)` bound to `viewer_projects`:
     - `system_id` / `investigation_id`: parse via `as_uuid`; `None` → `invalid_uuid_error`
       naming the field. Append `system_id = %s` / `investigation_id = %s`.
     - `state`: resolve `RunState(state)`; `ValueError` → `config_error_reason` with
       `ConfigErrorReason.INVALID_STATE` and `accepted_values=[s.value for s in RunState]`.
       Append `state = %s`.
     - On any filter error, return that `ToolResponse` (do not run the query).
  3. Clamp `limit` via `clamp_list_limit`. If `cursor`, decode via
     `decode_ts_uuid_cursor("runs.list", cursor)`; `InvalidCursor` →
     `invalid_cursor_error("runs")`. Append `(created_at, id) < (%s, %s)` bound to the
     decoded `(ts, uuid)`.
  4. Now short-circuit: if `viewer_projects` is empty, return an empty `ok` collection (no
     DB hit). Otherwise run the query:
     `SELECT * FROM runs WHERE {clauses} ORDER BY created_at DESC, id DESC LIMIT %s` with
     `limit + 1`, `dict_row` factory.
  5. `paginate(rows, limit)` → `(kept, truncated)`. `next_cursor =
     encode_ts_uuid_cursor("runs.list", kept[-1]["created_at"], kept[-1]["id"])` iff
     `truncated and kept`, else `None`.
  6. Render each kept row via `Run.model_validate(row)` → `envelope_for_run(run)` (no
     enrichment kwargs).
  7. Return `ToolResponse.collection("runs", "ok", items,
     suggested_next_actions=["runs.get", "runs.create"], data={"truncated": truncated,
     "next_cursor": next_cursor})`.

`Run.model_validate` must accept a `runs` row dict directly. The `RUNS` repository
declares `json_columns={"build_profile", "expected_boot_failure"}`; the raw `dict_row`
returns those columns as already-parsed JSON (jsonb) from psycopg, matching how
`systems.list` validates `System` from a raw row. Confirm in TDD; if the repo applies a
column transform on read, route the row through the repository's row adapter instead of
`Run.model_validate` directly.

### Registration (3-registration rule for a new tool)

1. `runs/registrar.py`: add `_register_runs_list(app, pool)` (called from `register`),
   exposing `name="runs.list"`, `annotations=_docmeta.read_only()`,
   `meta={"maturity": "implemented"}`, params `system_id` / `investigation_id` (open
   `str | None`), `state: RunState | None` (typed so the schema advertises the enum,
   matching `systems.list`'s `state`), `limit`, `cursor`. Docstring notes viewer gating +
   keyset pagination. Delegates to `list_runs` with `current_context()`.
2. `exposure.py`: add `"runs.list": _VIEWER` to `_TOOL_SCOPES` (the `runs` block).
3. `tests/mcp/core/test_tool_docs.py`: map `"runs.list"` to
   `("tests/mcp/lifecycle/test_runs_list.py",)`.

### Tests

New `tests/mcp/lifecycle/test_runs_list.py`, driving `list_runs` directly with an
injected pool (mirroring `test_systems_list.py`). Seed Runs by inserting their
prerequisite Investigation + (optional) System + Allocation + Resource, then
`RUNS.insert`. Cover:

- Lists the caller's Runs; excludes a terminal-vs-active mix? (no — all states list).
- Filter by `system_id` (only Runs on that System; an unbound Run with `system_id=None`
  is excluded when the filter is set).
- Filter by `investigation_id`.
- Filter by `state` (e.g. `running`) returns only matching Runs.
- Unknown `state` → `configuration_error` / `reason == "invalid_state"` /
  `accepted_values` contains a real `RunState`.
- Malformed `system_id` / `investigation_id` → `configuration_error` /
  `reason == "invalid_uuid"` / non-null `detail` echoing the bad id.
- No-leak: a Run in an ungranted project is omitted; a member project with no granted
  role is omitted; an empty-projects ctx returns an empty `ok` collection.
- Validation precedes scoping: an empty-projects ctx with a malformed `state` (or a
  malformed cursor) returns the `configuration_error`, not an empty collection — the error
  path must not depend on the caller's grants.
- A `failed` Run renders as a failure-shaped item carrying `failure_category` and no
  cross-project signal.
- Pagination: drains distinct timestamps; drains tied timestamps (id DESC tiebreaker);
  exactly-`limit` → `truncated is False` / `next_cursor is None`; malformed cursor →
  `configuration_error` / `reason == "invalid_cursor"`; `limit` clamps (0 → 1 row,
  10000 → all).
- Cap-after-filter: the oldest matching Run is not dropped by a small `limit` when many
  non-matching newer Runs exist (filter applied before the limit, as in
  `test_cap_applies_after_filters_no_undercount`).

## Acceptance (from the issue)

- "Runs on System X" and "Runs building in my project" each resolve in one call.
- Read-only and viewer-gated; cross-project Runs are not leaked.
- Uses the A3 (ADR-0192) pagination envelope.
