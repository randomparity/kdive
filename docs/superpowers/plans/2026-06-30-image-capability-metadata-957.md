# Image Capability Metadata Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make image capability metadata honest — one validated `Capability` vocabulary plus a thin computed-signal framework that degrades to `unverified` rather than lying (#957).

**Architecture:** Reframe `capabilities` as build-baked-tooling facts validated against a closed `Capability` StrEnum on all five carrier models; generalize the ADR-0253 kdump predicate into a `CapabilitySignal` registry surfaced as `images.describe` `data.capability_signals`; enforce honesty with a degrade-to-unverified invariant, a guarded `PLANNED_SIGNALS` backlog, and unit guards.

**Tech Stack:** Python 3.14, `uv`, Pydantic v2, `ruff`, `ty`, `pytest`. Commands via `just` (`just lint`, `just type`, `just test`, `just docs`, `just adr-status-check`).

## Global Constraints

- Ruff line length 100; lint set `E,F,I,UP,B,SIM`; zero warnings.
- `ty` runs whole-tree (src + tests) with strict defaults.
- Absolute imports only (`kdive.…`), no relative imports.
- Doc-style: plain, factual prose; never "critical"/"robust"/"comprehensive"/"elegant"/"Sprint".

- Conventional-commit subjects ≤72 chars; end every commit body with

```python
# src/kdive/images/capability_signals.py
"""Computed image-capability signals over build-recorded provenance (ADR-0286).

Generalizes the kdump-capability predicate: each signal computes a feature answer from a
build-recorded operand and degrades to a non-confident status when the operand is absent, so
metadata that predates a signal cannot report a confident-but-wrong answer.
"""

from __future__ import annotations

from collections.abc import Callable
from dataclasses import dataclass

from kdive.domain.catalog.images import Capability, ImageCatalogEntry
from kdive.images.kdump_support import KernelVersion, kdump_capability
from kdive.serialization import JsonValue

type SignalRender = Callable[[ImageCatalogEntry, KernelVersion], dict[str, JsonValue]]


@dataclass(frozen=True, slots=True)
class CapabilitySignal:
    """A computed capability answer over an image's build-recorded provenance.

    ``operand_keys`` are the provenance keys the build must record; when any is absent the
    ``render`` must return a non-confident status (never ``capable``).
    """

    name: str
    operand_keys: tuple[str, ...]
    render: SignalRender


@dataclass(frozen=True, slots=True)
class PlannedSignal:
    """A capability signal that is not honestly computable yet (its operand does not exist)."""

    name: str
    tracking_issue: str
    rationale: str


def render_kdump_signal(
    entry: ImageCatalogEntry, target_kernel: KernelVersion
) -> dict[str, JsonValue]:
    """The kdump capability block for ``entry`` against ``target_kernel`` (operand-driven)."""
    raw = entry.provenance.get("makedumpfile_version")
    cap = kdump_capability(
        makedumpfile_version=raw if isinstance(raw, str) and raw else None,
        target_kernel=target_kernel,
        kdump_tooling=Capability.KDUMP in entry.capabilities,
    )
    return {
        "makedumpfile_version": raw if isinstance(raw, str) else "",
        "target_kernel": cap.target_kernel,
        "capability": cap.status,
        "min_makedumpfile_required": cap.min_makedumpfile_required,
        "note": cap.note,
    }


KDUMP_SIGNAL = CapabilitySignal(
    name="kdump", operand_keys=("makedumpfile_version",), render=render_kdump_signal
)

REGISTERED_SIGNALS: tuple[CapabilitySignal, ...] = (KDUMP_SIGNAL,)

PLANNED_SIGNALS: tuple[PlannedSignal, ...] = (
    PlannedSignal(
        "sysrq", "#952", "SysRq availability can report false success; needs a build-recorded operand"
    ),
    PlannedSignal(
        "ssh_reachable", "#956", "sshd/keygen liveness is broken; not an honest per-image fact yet"
    ),
    PlannedSignal(
        "live_drgn",
        "#762/#697",
        "drgn liveness depends on provider introspection and profile ssh_credential_ref",
    ),
    PlannedSignal(
        "direct_kernel_bootable", "#954", "direct-kernel provisionability is discovered only by failure"
    ),
)

__all__ = [
    "REGISTERED_SIGNALS",
    "PLANNED_SIGNALS",
    "CapabilitySignal",
    "PlannedSignal",
    "render_kdump_signal",
]
```
