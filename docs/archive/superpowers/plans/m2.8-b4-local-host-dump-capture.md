## Task 1 — Refactor host-dump onto the KDUMP file-streaming path (TDD)

**Where it fits:** spec §1 + §4. Unifies the two capture methods on one file helper and deletes
the in-memory bytes host-dump path (and the dead bytes `extract_redacted` seam).

**Files:** `src/kdive/providers/local_libvirt/retrieve.py`,
`tests/providers/local_libvirt/test_retrieve.py`.

**Steps:**
1. **Failing test first.** Rewrite `test_capture_host_dump_uses_dump_seam` (and add siblings) so the
   fake `host_dump_capture` returns a **spooled `Path`** (reuse `_spooled_core`), and assert the
   HOST_DUMP path: (a) streams the raw core via `put_stream` under key `…/vmcore-host_dump`
   (`store.streams`, not `store.puts`); (b) persists `…/vmcore-host_dump-redacted` from the
   **file** dmesg seam (`extract_redacted_from_file`); (c) returns the file build-id and
   `raw_size_bytes == core.stat().st_size`; (d) removes the spool dir on success; (e) removes it on
   a `fail_on="vmcore-host_dump"` store failure (`INFRASTRUCTURE_FAILURE`); (f) `None` from the
   seam → `READINESS_FAILURE`. Run; confirm it fails because the current path uses bytes puts.
2. **Implement.** Extract a shared `_capture_via_file(system_id, method, core)` from `_capture_kdump`
   (stream raw, persist redacted-from-file, read build-id-from-file, `raw_size_bytes` from
   `core.stat().st_size`, `_remove_spool` in `finally`). Make both `_capture_kdump` and
   `_capture_host_dump` call it: kdump passes `self._wait_for_vmcore(system_id)`; host-dump passes
   `self._host_dump_capture(system_id)`. Both raise `_no_core` when the seam returns `None`.
3. **Change the seam type** `_HostDumpCapture` to `Callable[[UUID], Path | None]`. Drop the
   `extract_redacted` (bytes) constructor param + attribute and the `_real_extract_redacted_bytes`
   stub; update `from_env` to no longer pass it. Keep `read_vmcore_build_id` (bytes) — still used by
   `run_crash_postmortem`. Update the `capture` docstring (host-dump now also file-streamed).
4. Remove every `extract_redacted=` kwarg from the test constructors (`_kdump_retriever`,
   `_crash_retriever`, the inline ones).
5. Guardrails green; commit ("refactor(local): stream host-dump core from a worker temp file").

**Acceptance:** focused test green; no call site references the bytes `extract_redacted` seam;
`grep -n "_extract_redacted\b\|_real_extract_redacted_bytes" src tests` returns nothing.

## Task 2 — Implement the real `_real_host_dump_capture` seam (live_vm, pragma'd)

**Where it fits:** spec §2. The only new host-bound code; not CI-covered.

**Files:** `src/kdive/providers/local_libvirt/retrieve.py`.

**Steps:**
1. Replace the `MISSING_DEPENDENCY`-raising stub with the real seam, signature
   `_real_host_dump_capture(system_id: UUID) -> Path | None`, `# pragma: no cover - live_vm`:
   open `_libvirt_uri()`; `lookupByName(domain_name_for(system_id))` — a `libvirtError` (no such
   domain) returns `None`; `if not domain.isActive(): return None` (inactive → READINESS_FAILURE
   upstream); `mkdtemp(prefix="kdive-host-dump-")`, dump to `spool/vmcore` via
   `domain.coreDumpWithFormat(str(dest), libvirt.VIR_DOMAIN_CORE_DUMP_FORMAT_RAW,
   libvirt.VIR_DUMP_MEMORY_ONLY)`; on `libvirtError` `_remove_spool` + raise
   `INFRASTRUCTURE_FAILURE`; enforce `MAX_CORE_BYTES` on `dest.stat().st_size`, over-ceiling →
   `_remove_spool` + `CONFIGURATION_ERROR`; `conn.close()` in `finally`; return `dest`.
2. Mirror `_force_off_domain`/`_real_wait_for_vmcore` connection + cleanup discipline (close conn in
   `finally`, remove spool on every non-success path).
3. Guardrails green; commit ("feat(local): implement host-dump via libvirt domain core dump").

**Acceptance:** `ty`/`lint` green; the seam is `# pragma: no cover - live_vm`; coverage unchanged
in CI; manual read confirms every early-return/raise path removes the spool and closes the conn.

## Sequencing & rollback

Order 1→2→3→4 (Task 1 changes the seam type Task 2 implements; Task 2 must exist before the
descriptor advertises it; Task 4 is doc-only and last). Each task is its own commit so a `git
bisect` can pin any regression. Rollback = drop the offending commit; no migration, schema, or
external state to unwind. Full `just ci` before push.
